# 模拟交易模块需求文档

## 1. 需求概述

本系统是一个基于 Binance Depth WebSocket 数据流的模拟交易模块，允许用户实现自定义交易策略，并在模拟环境中执行交易，支持市价单和限价单的模拟撮合，同时提供策略盈亏跟踪和可视化功能。

## 2. 功能需求

| 需求编号 | 功能描述 | 需求来源 |
| :--- | :--- | :--- |
| REQ-001 | 实时接收 Binance Depth WebSocket 数据 | 输入是binance的depth websocket |
| REQ-002 | 提供交易策略接口定义 | 制定交易策略接口 |
| REQ-003 | 支持市价单下单 | 包括市价单 |
| REQ-004 | 支持限价单下单 | 和限价单 |
| REQ-005 | 模拟实时撮合成交 | 模拟交易模块根据实时行情模拟撮合成交 |
| REQ-006 | 返回订单状态 | 返回订单状态 |
| REQ-007 | 支持订单状态查询 | 系统必需功能 |
| REQ-008 | 支持账户资金管理 | 模拟交易必需功能 |
| REQ-009 | 跟踪策略盈亏 | 跟踪策略的盈亏 |
| REQ-010 | 绘制盈亏曲线 | 绘制盈亏曲线 |

## 4. 盈亏跟踪功能详细需求

### 4.1 盈亏计算方式

| 指标 | 计算方式 | 说明 |
| :--- | :--- | :--- |
| 已实现盈亏 (Realized PnL) | 卖出收益 - 买入成本 | 已平仓订单产生的盈亏 |
| 未实现盈亏 (Unrealized PnL) | 当前持仓市值 - 持仓成本 | 未平仓订单产生的浮动盈亏 |
| 总盈亏 (Total PnL) | 已实现盈亏 + 未实现盈亏 | 账户总体盈亏情况 |
| 收益率 (Return Rate) | 总盈亏 / 初始资金 × 100% | 相对收益百分比 |

### 4.2 盈亏数据记录

| 数据项 | 描述 | 更新频率 |
| :--- | :--- | :--- |
| timestamp | 记录时间戳 | 每次行情更新 |
| totalPnl | 总盈亏 | 每次行情更新 |
| realizedPnl | 已实现盈亏 | 订单成交时 |
| unrealizedPnl | 未实现盈亏 | 每次行情更新 |
| equity | 账户权益 (余额 + 未实现盈亏) | 每次行情更新 |

### 4.3 盈亏曲线数据点

每个数据点包含：
- 时间戳
- 总盈亏值
- 账户权益
- 收益率

### 4.4 可视化需求

| 需求 | 描述 |
| :--- | :--- |
| 实时更新 | 每次行情更新时刷新曲线 |
| 时间范围 | 支持选择不同时间范围（日/周/月/全部） |
| 对比基准 | 可叠加显示资金曲线作为对比基准 |
| 关键指标 | 显示最大回撤、平均收益率、夏普比率等 |

## 5. 数据字典

| 字段名 | 类型 | 描述 | 约束 |
| :--- | :--- | :--- | :--- |
| symbol | string | 交易对，如 BTC/USDT | 必填 |
| orderId | string | 订单唯一标识 | 系统生成 |
| orderType | enum | LIMIT/MARKET | 必填 |
| side | enum | BUY/SELL | 必填 |
| price | decimal | 限价单价格 | 限价单必填 |
| quantity | decimal | 下单数量 | 必填 |
| status | enum | NEW/FILLED/CANCELLED/PARTIALLY_FILLED | 系统维护 |
| filledQuantity | decimal | 已成交数量 | 系统维护 |
| timestamp | long | 订单创建时间戳 | 系统生成 |
| bids | array | 卖盘深度数据 | WS推送 |
| asks | array | 买盘深度数据 | WS推送 |

## 6. 非功能需求

| 需求类型 | 描述 |
| :--- | :--- |
| 性能需求 | 行情数据处理延迟 < 10ms |
| 可扩展性 | 支持多策略同时运行 |
| 数据持久化 | 订单和盈亏记录可持久化存储 |
